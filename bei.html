Barely Enough Ingredients

A recipe suggestion website that stores a database of the clients pantry and gives them ideas of what to cook using user data.

Final Release

December 8, 2017 by Ian

The current version of Barely Enough Ingredients can be cloned via the repo and run on a computer with ruby installed.

You can view our documentation here.

I worked on this webapp with Ryan Manske, Anish Ravi, and Meng Jie Ye as our project for Software Design and Documentation, a class at RPI.

We surveyed campus to figure out what features in an ingredient/recipe database our user base would be interested in having, and found out that being able to rate recipes and be recommended new foods based on several criteria (does the user have the ingredients, what is the cost to cook this meal, is it related to similar foods this user has rated highly in terms of taste, cuisine, etc.) was most important.

Our front end design plans changed drastically from one iteration to the next, ultimately ending up with a very minimalistic design, as we did not have time to complete the graphics we originally set out to create. Here you can see the progression from paper sketches, to photoshop design planning, to the concluding ruby design using bootstrap.

We also decided to call the actual webapp MOM – Many Online Meals (I asked permission from my mom before we decided to use this name, as we did not want any copyright problems once we blew up on the stock market).

We utilized ruby on rails to create the responsive html pages, with bootstrap. For the recipes themselves, we integrated the Spoonacular API, which has thousands of statistics, such as cook time, user popularity, and average price per ingredient. In terms of group work, we decided to use Scrum for our software development project management framework, which worked really well and made efficient use of our time.

There is no plan to continue the project as of now, however, I may pick it up again and finish the last couple features we planned to implement and fix the remaining bugs.
