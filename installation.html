Interactive Sound Installation

February 1st, 2018 by Ian

I began work conceptualizing a semester long multichannel project. Currently the plan is to create ambisonic recordings and learn the basics of multichannel music and sound production. I plan to then incorporate this new material in some sort of interactive sound installation that lets users modify, learn, and play with the sounds and virtual space they hear in the room.

In order to complete this task, I will create a webapp for people to access the ambisonic array in the room they are currently in via a wireless connection to a server. Whoever sets up the array will also set up the server and be able to specify the location of each speaker among other ambisonic requirements for room space and acoustics. A user of the webapp will be able to access the page on their mobile device or tablet/laptop computer, as well as any other device that can connect to the internet and access the web page where the app is being hosted with the correct software implementations - i.e., Internet Explorer and Java or Google Chrome and Adobe Flash.

Next, since I would like this program to be more of an educational tool than anything, I plan to let users be able to experiment with the sounds and waveforms without necessarily needing to be connected to the server, so I will have an option to output audio directly from their device. It will be interesting to see in large groups of people, the methods used for sound grouping socially and the effects of sounds populated in large clusters and so forth.

Here are some conceptualization sketches I made today for the user interface. I plan to use bootstrap for dynamic webpages, and JavaScript for designing interfaces. As of right now, the idea for the sounds is night creatures, specifically bugs like crickets and katydids.

I am hoping to get the webapp finished for testing within a month. I plan to use Ruby on Rails as of right now, but that might change, as I am currently learning node.js. We will see.
